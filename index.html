What this is

A from-scratch recreation by Jetha Chan (@jetha) using Grok 4.5, driven by Abby’s public footage, her technique write-up, and iterative visual fixes. Loop: cocoon → mid → moth → full noisy center-to-edge wipe (moth design rotates each cycle). Animation lives in the fragment shader (iTime / iResolution only — Shadertoy-style).

What Grok got right

  • Procedural mandate — no textures; pure math so it can boot before assets exist.
  • 2D SDFs for “pixels” — rounded rects, registration marks, AA coverage (Abby + Book of Shaders / Íñigo Quílez lineage).
  • Bit-packed moths — 21×21 stages sampled from stable hold frames and stored as uint row masks (Wolfie’s designs).
  • Per-cell decision — position + shape + time + noise, not a scripted sprite sheet.
  • Center-out arrival — new lit pixels form at the center and ripple outward to build the glyph (after rewatching footage).
  • Lime as a wave front — energy rides an expanding ring; full on the silhouette, weaker past the glyph bounds.
  • Timeline in the shader — form cocoon → mid → moth → dissolve → empty; moth variants cycle. JS only feeds time.

What Grok got wrong

  • Vertical lime “scan line” — first pass treated lime as a top→bottom wipe. Footage (and Abby’s model) is a radial ripple from the grid center.
  • Starting mid-hold — opened on a fully formed moth. Real flow starts empty and grows the first glyph from the center.
  • Falloff extremes — empty-grid lime was either a full unmasked ring or crushed to almost nothing. Needed a ~2–3 cell halo past the glyph, not a hard mask and not a free disc.
  • Fake wipe sparkles — invented 2×2 corner lime (and earlier full edge greens) after dissolve. Looked “registration-y” but wasn’t faithful; removed.
  • JS stage machine — early versions drove shape locks and phases from JavaScript. Reworked so the timeline is pure GLSL.
  • Not pixel-perfect — bitmasks are sampled from compressed video, not extracted from Bungie’s binaries. Close, not identical.

How to poke it

Open the Shader tab. Edit the GLSL, hit Compile (or Ctrl+Enter). Errors show under the editor without killing the last good program.
